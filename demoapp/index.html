Process

How It Works

Seven phases from zero infrastructure to a fully automated disaster recovery system with auto-failover.

01
Build the Clusters
Two EC2 instances launched via AWS CLI. Kubeadm initializes K8s on each. Flannel provides pod networking across both regions.
KUBEADM + EC2
02
S3 + CRR Setup
Primary S3 bucket in us-east-1 with versioning. Replica in us-west-2. Cross-Region Replication copies every object automatically within seconds.
S3 + IAM
03
Install Velero
Velero v1.13.2 installed on both clusters. Primary writes backups. Standby reads replica in ReadOnly mode. IAM least-privilege enforced.
VELERO v1.13.2
04
Schedule Backups
Velero Schedule CR triggers full cluster backup every 24h at 02:00 UTC. All namespaces, resources, PVC snapshots. 7-day retention.
CRON + VELERO
05
Deploy Website
Nginx serves this HTML page as a K8s Deployment on BOTH clusters. NodePort exposes port 80. Same site runs in both regions simultaneously.
NGINX + K8S
06
Route53 Failover
Health check pings primary every 30s. If 3 checks fail, DNS auto-switches to failover region. Zero manual intervention required.
ROUTE53
07
Measure RTO
restore-test.sh runs weekly via cron. Deletes namespace, triggers Velero restore, polls until complete, logs exact time taken.
BASH + CRON
08
Auto Recovery
Primary goes down → Route53 switches DNS → Failover serves website → Velero restores latest backup → Full DR complete.
FULL DR
09
Weekly Drill
Automated restore drills every Monday generate RTO reports. Proves the system works before a real disaster ever hits.
AUTOMATION
